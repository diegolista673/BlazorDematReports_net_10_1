## STATI HANGFIRE SEMPLIFICATI - COMPLETATO ##
Gli stati dei task Hangfire sono stati semplificati e documentati nel tab Monitoring.

STATI RIMOSSI: SUCCESS, STARTED, RUNNING, STOPPED, PAUSED, IMPORTED

STATI MANTENUTI (3 stati essenziali):
  1. CONFIGURED (Blu/Info): Task configurato, pronto ma non ancora eseguito
     - Impostato quando: il task viene creato per la prima volta
  
  2. COMPLETED (Verde/Success): Ultima esecuzione completata con successo
     - Impostato quando: il job Hangfire termina senza eccezioni
     - Il task continua ad essere schedulato se Enabled = true
  
  3. ERROR (Rosso/Error): Ultima esecuzione fallita con errore
     - Impostato quando: il job Hangfire genera un'eccezione
     - Il task continua ad essere schedulato se Enabled = true
     - Dettagli errore salvati nel campo LastError

CICLO DI VITA DEI TASK:
  Creazione → CONFIGURED (Enabled=true)
       ↓
  Hangfire esegue secondo CRON
       ↓
  ├─ Esecuzione OK → COMPLETED (rimane schedulato)
  │     ↓
  │  Prossima esecuzione → mantiene COMPLETED o va in ERROR se fallisce
  │
  └─ Esecuzione KO → ERROR (rimane schedulato)
        ↓
     Prossima esecuzione → mantiene ERROR o va in COMPLETED se ha successo

DASHBOARD MONITORING SEMPLIFICATO:
  
  KPI Cards (3 card):
    - Fasi Totali: numero di fasi configurate
    - Task Totali: numero totale di task nel sistema
    - Servizi Mail: task con MailServiceCode configurato
  
  Distribuzione Stati:
    - Grafico con CONFIGURED, COMPLETED, ERROR
  
  Distribuzione Tipi:
    - Grafico con Giornaliero, Temporizzato, Mensile
  
  DataGrid Fasi (5 colonne):
    | Fase | Task Totali | Configured | Completed | Error | Data Reading |
    

    - Ogni colonna mostra conteggio task per stato
    - Tooltip su ogni header per spiegare cosa rappresenta

NOTA IMPORTANTE:
  - Tutti i task con Enabled = true vengono eseguiti automaticamente
  - Gli stati CONFIGURED, COMPLETED, ERROR indicano solo lo stato corrente
  - Il flag Enabled controlla se il task viene eseguito o meno
